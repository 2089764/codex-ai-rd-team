# 评审基线（稳定文档）

> 文档类型：Review Checklist  
> 用途：代码评审与发布前自检基线

## 1. 功能正确性

- [ ] 需求是否有对应实现（不多不少）
- [ ] 主路径是否可运行
- [ ] 边界条件是否有明确处理

## 2. 架构一致性

- [ ] 是否遵循 `orchestrator/` 单一执行内核原则
- [ ] 是否避免在 `rd-team/` 复制业务逻辑
- [ ] 是否与现有模块职责一致

## 3. 可测试性

- [ ] 新增/修改行为是否有测试覆盖
- [ ] 关键回归路径是否被验证
- [ ] 测试命令是否可复现

## 4. 可维护性

- [ ] 命名与结构是否清晰
- [ ] 异常处理是否明确
- [ ] 文档是否同步更新（README/ARCHITECTURE/docs）

## 5. 运行产物边界

- [ ] 自动产物是否落在 `runtime/artifacts/<run_id>/`
- [ ] 稳定文档是否未被运行覆盖
- [ ] `runtime/run-*.json` 是否可用于排障回放

## 6. 结论模板

```text
APPROVE: <简要结论>
```

或

```text
REJECT: <问题摘要 + 建议修复点>
```
