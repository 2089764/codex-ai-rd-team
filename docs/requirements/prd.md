# PRD（稳定文档）

> 版本：v0.1  
> 状态：Active  
> 适用范围：`codex-ai-rd-team` 编排内核与 A1 兼容层

## 1. 背景与目标

当前团队需要一个可复用的 RD 编排内核，将自然语言目标转为可执行的角色工作流，并可追踪状态、产物与回退闭环。

目标：
- 统一入口：通过 CLI/兼容命令触发编排
- 统一内核：业务逻辑只在 `orchestrator/` 实现
- 统一追踪：每次运行可回放、可审计、可定位失败点

## 2. 目标用户

- 新加入项目的工程师（快速上手）
- 团队内部开发者（执行与维护）
- 外部评审者（架构理解与专业展示）

## 3. 核心能力范围

### In Scope
- 任务模式判定（4 模式）
- 角色链编排（按模式 + `has_frontend`）
- 双执行后端（Codex/Echo）
- reviewer/tester 反馈闭环与重试策略
- 运行态与执行产物落盘
- A1 兼容目录映射

### Out of Scope（当前版本）
- 分布式调度与跨节点容错
- 实时可视化 UI
- 多租户权限系统

## 4. 成功标准（验收）

- CLI 能完成一次 end-to-end 编排，输出 `status=completed`
- `runtime/run-*.json` 可还原该次运行上下文
- 自动产物写入 `runtime/artifacts/<run_id>/...`
- reviewer/tester 负反馈能触发重试与终态分流
- 全量单测通过（当前 86 项）

## 5. 非功能要求

- 可维护：模块职责清晰，边界稳定
- 可扩展：新增 profile/role 不破坏主流程
- 可测试：关键模块具备单元/集成测试
- 可追溯：错误与状态变化有落盘证据
